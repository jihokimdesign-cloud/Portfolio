-- An accessibility tool that wasn't accessible. I rebuilt its foundations.

Project Sidewalk is an open-source civic platform that crowdsources urban accessibility data to train machine learning models. With over 350,000 labels collected, it had proven its value — but the architecture and experience underneath were brittle. I architected the zero-defect, WCAG 2.2 AA design system now adopted across three cross-functional teams.

### 01 — Diagnose with data, not intuition

Hotjar heatmaps and session recordings pinpointed drop-off moments; GitHub issues and forum threads surfaced recurring pain; a heuristic WCAG evaluation scored the legacy interface. The behavioral data was decisive — **68% of clicks concentrated on the map center** while labels outside the viewport went undiscovered, and **42% of users abandoned the sidebar controls** after one attempt.

### 02 — Tokens as a single source of truth

I mapped Figma variables 1:1 with the engineers' SCSS variables — `color.primary` to `$color-primary`, `spacing.md` to `$spacing-md` — eliminating guesswork during handoff.

### 03 — A working contract, not documentation

A design system is only as successful as its adoption rate. Working tightly with one PM and two developers, I ran phased rollouts with comprehensive spec handoffs — treating the system as a contract between design and engineering rather than a style guide nobody reads.

<SectionTitle num="05" kicker="Reflection" title="Systems thinking in civic tech" />

-- A design system is not a style guide — it's a tool for empowerment.

This project shifted my perspective from designing for the happy path to designing for resilience in open-source and civic environments. When inclusivity lives in the system architecture, every new feature inherits those values — accessibility becomes automatic instead of an afterthought.
